14.TYPES OF XSS

There are several types of Cross-Site Scripting (XSS) attacks:

1. Stored XSS (Persistent XSS)

- Malicious code is stored on the server and executed when a user views the affected page.

- Can be injected through user-generated content, such as comments or forum posts.

2. Reflected XSS (Non-Persistent XSS)

- Malicious code is reflected off the server and executed on the user's browser.

- Typically occurs when user input is not properly sanitized or validated.

3. DOM-based XSS

- Malicious code is executed in the browser's DOM (Document Object Model) without any server-side interaction.

- Can occur when client-side JavaScript code is vulnerable to injection attacks.

4. Self-XSS

- A type of XSS attack that requires user interaction to execute malicious code.

- Often used in phishing attacks to trick users into executing malicious code.

5. Blind XSS

- A type of XSS attack where the attacker does not have direct access to the vulnerable page.

- Can be used to attack administrators or other users who have access to the vulnerable page.
